### PASO 3: Editar el archivo de migración generado

**¿POR QUÉ?** Alembic detecta tablas y columnas automáticamente, pero NO detecta:
- ENUMs de PostgreSQL (ya fueron creados en la migración anterior)
- Funciones de PL/pgSQL
- Triggers

Como los ENUMs ya fueron creados en la migración anterior (`genero_enum`, `estado_empleado_enum`),
solo necesitamos agregar la función `fn_set_updated_at()` y los triggers.

**Abre el archivo generado** (ej: `alembic/versions/xxxxx_agregar_tabla_usuario.py`) y **agrega esto al inicio de la función `upgrade()`**:

```python
def upgrade() -> None:
    # ====================================================================
    # FUNCIÓN: Actualizar automáticamente el campo updated_at
    # ====================================================================
    op.execute("""
    CREATE OR REPLACE FUNCTION fn_set_updated_at()
    RETURNS TRIGGER AS $$
    BEGIN
        NEW.updated_at = CURRENT_TIMESTAMP;
        RETURN NEW;
    END;
    $$ LANGUAGE plpgsql;
    """)

    # ====================================================================
    # TRIGGERS: Aplicar fn_set_updated_at a todas las tablas con updated_at
    # ====================================================================
    tablas_con_updated_at = [
        'rol', 'departamento', 'cargo', 'empleado', 'usuario'
    ]

    for tabla in tablas_con_updated_at:
        trigger_name = f'trg_{tabla}_set_updated_at'
        op.execute(f"""
        CREATE TRIGGER {trigger_name}
        BEFORE UPDATE ON rrhh.{tabla}
        FOR EACH ROW
        EXECUTE FUNCTION fn_set_updated_at();
        """)

    # ### commands auto generated by Alembic - please adjust! ###
    # (aquí sigue el código generado por Alembic)
    ...
```

**Y agrega esto al inicio de la función `downgrade()`** (para poder revertir la migración si es necesario):

```python
def downgrade() -> None:
    # ====================================================================
    # ROLLBACK: Eliminar triggers y función
    # ====================================================================
    tablas_con_updated_at = [
        'rol', 'departamento', 'cargo', 'empleado', 'usuario'
    ]

    for tabla in tablas_con_updated_at:
        trigger_name = f'trg_{tabla}_set_updated_at'
        op.execute(f'DROP TRIGGER IF EXISTS {trigger_name} ON rrhh.{tabla};')

    op.execute('DROP FUNCTION IF EXISTS fn_set_updated_at();')
```

## ❓ ¿Problemas?

### Error: "relation already exists"
**Causa:** Ya aplicaste esta migración antes.
**Solución:** Verifica con `python -m alembic current` qué versión tienes aplicada.

### Error: "No such file or directory: alembic.ini"
**Causa:** No estás ejecutando desde la raíz del proyecto.
**Solución:** cd al directorio raíz (donde está el archivo `alembic.ini`).

### Error: "Can't locate revision identified by 'xxxx'"
**Causa:** Alembic perdió la sincronización con la BD.
**Solución:** Ejecuta `python -m alembic stamp head` para sincronizar.
